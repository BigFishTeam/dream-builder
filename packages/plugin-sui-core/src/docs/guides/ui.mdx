- 是/否状态只需要：通过和不通过`两种`色彩即可。多彩的状态繁琐易混乱。
- 关于`查询统计类业务的多个模块之间的布局`，sui建议 -- 
① 模块竖向排列
② 模块可以使用`小标题`（caption组件）来标识单独模块的含义
③ echart图表内不展示标题，而是用`小标题`来代替
④ `小标题`只有在需要解释该模块的含义的时候才使用。对于单一查询结果的信息展示模块（显而易见的含义），省略小标题即可。
⑤ 竖向排列的组件之间的距离是50px（上组件和下组件可视元素之间的距离）；
- `chart`图表类，sui建议：
① 左右100%布局，高度固定像素布局
② labal标签顶部居中展示
③ 记得引用sui提供的`echart主题`哦
