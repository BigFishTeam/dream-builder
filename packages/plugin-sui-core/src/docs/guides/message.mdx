## 信息架构
`以下是一些经验的积累和沉淀`
只对相关问题进行意见和建议，不对业务约束。
<a href="https://git.code.oa.com/comment_frontend/sui-core/wikis/Standard" target="_blank"  >最新地址</a>


- 关于`系统架构`：
① sui提供两种布局风格：`首页`、`白色内容区+灰色背景区`；
② 现有的sui布局方式(`白色内容区+灰色背景区`)的风格简约、清爽、聚焦。特别适合导航、内容少的中小系统。
③ 对于组织大型系统or平台，建议采用`首页+子系统`的组合方式。首页作为入口的集合，子系统承载功能。

- 关于`导航`：
导航承载`相对固定`、不经常改变的内容路由。如果子路由较多(>10)的情况or经常变动(增减)的情况，可以聚合到一个模块内采用“栏目+子栏目”的方式去解决。

- 关于`帮助说明？`：
① 表格类查询的结果要加上“？”的提示和对表格内字段的`解释说明`。
② `？帮助说明`功能的位置应该放在页面固定的几个地方：① 小标题caption ② table右上角。
③ 不好的体验和设计：散落在无规律的地方、一个页面过多的出现。
